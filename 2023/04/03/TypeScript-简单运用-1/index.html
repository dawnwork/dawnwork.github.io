TypeScript 简单运用 (一)

TypeScript 是以 JavaScript 为基础的构建语言， 一个 JavaScript 的超集，扩展了 JavaScript 并添加了类型，可以在任何支持 JavaScript 的平台中执行

与 JavaScript 不同的是，TypeScript 强制类型和静态类型检查。这意味着，编写代码时必须指定变量的类型，并在编译时检查这些类型。这样做可以帮助开发人员在代码执行之前发现潜在的问题，降低了调试和错误修复的成本。TypeScript
还提供了对 ES6、ES7 和 ES8 新特性的支持，这些特性可帮助开发人员编写更简洁、更优雅的代码。

TypeScript 的优势

  • 类型

    • 在 JavaScript 中，由于变量的类型是动态的，因此很难在开发过程中识别出类型错误。而在 TypeScript 中，开发人员必须显式地指定变量的类型，并在编译时检查这些类型。

  • 新特性支持

    • TypeScript 支持 ES6、ES7 和 ES8 的新特性，这些特性可以帮助开发人员编写更简洁、更优雅的代码。

  • 添加新特性

    • TypeScript 也添加了许多 ES 没有的新特性，这些特性使得 TypeScript 更加适合大型应用程序的开发。例如，TypeScript
      支持接口、泛型、枚举、命名空间、装饰器等特性。

注： TypeScript 不能被 JavaScript 解析器直接执行，需要先通过编译器将 TypeScript 转换为 JavaScript

总结

TypeScript 增加了类型系统和静态类型检查，支持 ES6、ES7 和 ES8 的新特性，添加了许多 ES 没有的新特性

类型注解

引用类型

在 TypeScript 中，引用类型是指除了原始类型（如数字、字符串、布尔值、空值等）之外的所有类型，它们都是通过对象来表示的，包括数组、函数、对象等。

数组类型

数组类型表示一个由相同类型的元素组成的数组，有两种写法：

  1. 使用后缀 []，例如 number[] 表示由数字类型组成的数组。
  2. 使用泛型数组类型 **Array<元素类型>**，例如 Array 表示由字符串类型组成的数组。 另外，如果数组中有多种类型，可以使用联合类型来表示，例如 (number | string)[] 表示数组中可以包含数字类型和字符串类型的值。
元组 Tuple

元组是另一种类型的数组，它可以确切的知道包含多少个元素，以及特定索引元素的类型

函数类型

函数类型表示一个函数的参数和返回值的类型，使用 (参数列表: 参数类型) => 返回值类型 的格式来表示。

例如，一个加法函数 add 的类型为 (num1: number, num2: number) => number，表示它有两个参数，都是数字类型，返回值也是数字类型。

对象类型

对象类型表示一个包含多个属性的对象，每个属性都有自己的名称和类型，使用 {属性名: 属性类型, …} 的格式来表示，例如 {name: string, age: number} 表示一个包含 name 和 age 两个属性的对象，其中 name 的类型为字符串类型，age 的类型为数字类型。

需要注意的是，在 TypeScript 中对象类型的属性都是必需的，也就是说如果我们定义了一个对象类型，那么在创建该对象时必须为该对象的所有属性赋值，否则编译时就会报错。如果我们希望某个属性是可选的，可以使用 ? 符号来表示，例如 {name?: string, age?: number} 表示这个对象可以包含 name 和 age 属性，也可以不包含它们。

另外，如果一个对象的属性类型是不确定的，可以使用索引签名来表示，使用 [属性名: 属性类型]: 值类型 的格式来表示，例如 {[propName: string]: any} 表示这个对象可以包含任意属性名称和值类型的属性。

JS 中对象就是由属性和方法构成的，而 TS 中对象类型就是用于描述对象的结构的，对象有什么属性和方法
直接采用 {} 来描述对象结构。属性采用 属性名: 类型；方法采用 方法名(): 返回值类型 的形式

总结

类型注解是 TypeScript 的一个强大特性，它可以帮助我们在编写代码时尽早发现类型错误，从而提高代码的可靠性和可维护性。 :类型 就是类型注解 例如： :number
