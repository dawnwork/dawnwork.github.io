Git简单上手

Git 是一个免费的、开源的分布式版本控制系统，它被设计用来管理软件开发过程中的各种变更。通过使用
Git，你可以轻松地记录和跟踪项目中的所有更改，包括代码的修改、文件的添加、删除等，同时也能够协同工作，避免不同版本之间的代码冲突。

文件状态

在 Git 中，每个文件都有一个状态，这个状态告诉你这个文件在 Git 中的状态。Git 文件状态分为三种：未跟踪、已暂存和已提交。

  • 未跟踪（Untracked）：表示这个文件没有被 Git 管理，也没有被添加到 Git 仓库中。在使用 git status 命令查看文件状态时，未跟踪的文件会被列出。

你可以使用 git add 命令将这个文件添加到 Git 仓库中，Git 会将这个文件标记为已暂存状态。

  • 已暂存（Staged）：表示这个文件已经被添加到 Git 仓库中，但是还没有被提交。在使用 git status 命令查看文件状态时，已暂存的文件会被列出。

你可以使用 git commit 命令将已暂存的文件提交到 Git 仓库中，Git 会将这个文件标记为已提交状态。

  • 已提交（Committed）：表示这个文件已经被提交到 Git 仓库中，你可以在 Git 历史记录中查看这个文件的所有更改记录。在使用 git
    status 命令查看文件状态时，已提交的文件不会被列出。

除了上述命令外，还有一些其他的命令和操作可以管理和处理文件状态，包括：

  • git restore <文件名>：将文件恢复到最新的已提交状态或已暂存状态。
  • git restore --staged <文件名>：取消已暂存文件的暂存状态，使其恢复到未暂存状态。
  • git rm <文件名>：将文件从 Git 仓库中删除，如果文件被修改过，需要使用 -f 选项强制删除。
  • git mv <源文件名> <目标文件名>：将文件移动或重命名，并自动将修改添加到 Git 暂存区中。

Git 分支

在 Git 中，每一次代码提交都会创建一个节点，用来记录代码当前状态。这些节点会构成一个树状结构，而树状结构就意味着这个树会存在一个分支。默认情况下，Git
仓库只有一个主分支，即 master 分支。它允许开发人员在不破坏主干代码的情况下，创建自己的代码分支，进行实验性的开发和测试。在开发过程中，使用分支可以更好地管理代码，同时也可以允许多人同时开发而不会干扰彼此的工作。

变基

变基（rebase）是 Git 中的一个强大工具，它可以让你修改提交记录的历史，并让提交记录更加整洁、清晰。变基操作会将当前分支的提交记录在另一个分支的基础上重新播放一遍，从而达到修改提交历史的目的。

变基操作的优点在于它可以减少合并提交，从而使提交记录更加清晰。在某些情况下，合并提交可能会使提交历史变得混乱不堪，而变基操作可以避免这种情况的发生。例如，在多人协作的项目中，如果每个人都在自己的分支上进行开发，那么在将这些分支合并到主分支之前，可以使用变基操作将这些分支的提交记录整理成一个线性的提交历史。

Git 远程仓库
